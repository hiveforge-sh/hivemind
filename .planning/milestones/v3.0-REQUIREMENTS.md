## v3.0 Requirements

### Setup Wizard

- [x] **INIT-01**: User can run `npx hivemind init` to start interactive setup
- [x] **INIT-02**: Wizard prompts for vault path with path validation
- [x] **INIT-03**: Wizard offers template selection (worldbuilding, research, people-management, custom)
- [x] **INIT-04**: Wizard auto-detects vault structure and suggests appropriate template
- [x] **INIT-05**: Wizard generates config.json with selected settings
- [x] **INIT-06**: Wizard outputs Claude Desktop config snippet for easy MCP setup
- [x] **INIT-07**: Non-interactive mode via `--config <path>` flag for CI/automation

### CLI Validation

- [x] **VALD-01**: `validate` command scans entire vault for files with missing frontmatter
- [x] **VALD-02**: `validate` reports files with invalid frontmatter (wrong type, missing required fields)
- [x] **VALD-03**: `validate` uses template schemas for field validation
- [x] **VALD-04**: `validate` returns exit code 0 for success, 1 for validation errors, 2 for config errors
- [x] **VALD-05**: `validate --json` outputs machine-parseable results

### CLI Fix

- [x] **FIX-01**: `fix` command shows dry-run preview by default (no file modifications)
- [x] **FIX-02**: `fix --apply` flag required to actually modify files
- [x] **FIX-03**: `fix` uses folder-to-type mapping from config to determine entity type
- [x] **FIX-04**: `fix` prompts for type when folder mapping is ambiguous
- [x] **FIX-05**: `fix --yes` flag skips confirmation prompts for automation
- [x] **FIX-06**: `fix --json` outputs machine-parseable results

### Folder Mapping

- [x] **FMAP-01**: Config supports folder-to-type mapping (e.g., "People/" -> character)
- [x] **FMAP-02**: Folder mapping used by both CLI and Obsidian plugin
- [x] **FMAP-03**: Multiple folders can map to same type (e.g., "Characters/", "People/" -> character)

### Obsidian Commands

- [x] **OBSD-01**: "Hivemind: Add frontmatter" command inserts template for current file
- [x] **OBSD-02**: Add command auto-detects type from folder mapping
- [x] **OBSD-03**: Add command shows type selection modal when mapping is ambiguous
- [x] **OBSD-04**: "Hivemind: Validate" command checks current file against schema
- [x] **OBSD-05**: Validate command shows clear feedback (valid/invalid with specific issues)
- [x] **OBSD-06**: "Hivemind: Fix" command shows missing fields and offers to add them
- [x] **OBSD-07**: Fix command uses modal UI for field-by-field review

### Error Handling

- [x] **ERR-01**: Clear error message when config.json is missing
- [x] **ERR-02**: Clear error message when vault path is invalid
- [x] **ERR-03**: Helpful suggestions when vault has no valid entities
- [x] **ERR-04**: Platform-agnostic path handling (Windows/Unix)
